# Issue 007: Admin Courier Estimate Cost 404

## Reported
2026-07-06

## Summary
Clicking "Calculate Total COD" on the admin order edit page returns `404 Not Found`.

**Request:**
```
POST http://localhost:3000/admin/orders/R462343273/courier/estimate_cost 404
```

## Root Cause
The inline JavaScript fetch URLs had the `courier` and `orders` path segments in the wrong order.

**JS called (wrong):**
```
/admin/orders/:id/courier/estimate_cost
/admin/orders/:id/courier/confirm
```

**Routes actually define:**
```ruby
namespace :admin, path: Spree.admin_path do
  resources :orders, only: [] do
    scope 'courier', controller: '/spree/pathao_courier/admin/courier', as: 'courier' do
      post 'estimate_cost'
      post 'confirm'
      get  'tracking'
    end
  end
end
```

**Actual URLs generated by Rails:**
```
POST /admin/courier/orders/:order_id/estimate_cost
POST /admin/courier/orders/:order_id/confirm
GET  /admin/courier/orders/:order_id/tracking
```

The `scope 'courier'` inside `resources :orders` nests the path as `/admin/courier/orders/:order_id/...`, not `/admin/orders/:order_id/courier/...`.

## Fix
Updated both fetch URLs in `_tracking_section.html.erb`:

```diff
- fetch('/admin/orders/' + ORDER_ID + '/courier/estimate_cost', {
+ fetch('/admin/courier/orders/' + ORDER_ID + '/estimate_cost', {

- fetch('/admin/orders/' + ORDER_ID + '/courier/confirm', {
+ fetch('/admin/courier/orders/' + ORDER_ID + '/confirm', {
```

**File changed:** `backend/engines/spree_pathao_courier/app/views/spree/pathao_courier/admin/courier/_tracking_section.html.erb`

## Verification
- `docker compose exec web bundle exec rails routes | grep courier` confirms URLs
- Server restarted after fix
- Tested: Calculate Cost and Confirm buttons now hit correct endpoints

## 500 Error: AddressNotFoundError + TypeError (2026-07-06)

### Symptom
After fixing the 404 routing issue, clicking "Calculate Total COD" now hits the endpoint but returns `500 Internal Server Error`.

**Server log:**
```
NameError (uninitialized constant Spree::PathaoCourier::AddressNotFoundError)
Caused by: TypeError (no implicit conversion of String into Integer)
```

### Root Cause — Two Bugs

#### Bug A: Missing `AddressNotFoundError` class
The controller (`courier_controller.rb:31`) rescues `Spree::PathaoCourier::AddressNotFoundError`, and the `AddressResolver` raises it, but the class is never defined in `error.rb`.

**`error.rb` only defines:**
```ruby
class Error < StandardError; end
class AuthenticationError < Error; end
class ApiError < Error; end
class ShippingError < Error; end
```

**Missing:** `class AddressNotFoundError < Error; end`

When the TypeError occurs, Ruby tries to match against the rescue clause. Resolving the undefined constant raises `NameError`, which masks the original `TypeError`.

#### Bug B: String-to-Integer TypeError
The Pathao API returns IDs as strings (e.g., `city_id: "123"` instead of `123`). These string IDs flow through:
1. `AddressResolver#resolve_city` → `city['city_id']` (string)
2. `AddressResolver#resolve_zone(city_id)` → used in URL interpolation (works)
3. `AddressResolver#resolve_area(zone_id)` → used in URL interpolation (works)
4. `CheckoutService#estimate_cost` → `address_data[:zone_id]` passed to `CostEstimator` (string)
5. `CostEstimator#call` → `recipient_zone` sent to Pathao price-plan API as string

The Pathao price-plan API or internal processing expects integer zone IDs, causing `TypeError (no implicit conversion of String into Integer)`.

### Fix Plan
1. **Add `AddressNotFoundError`** to `backend/engines/spree_pathao_courier/app/services/spree/pathao_courier/error.rb`
2. **Add `.to_i` conversions** in `AddressResolver` when extracting IDs from API responses:
   - `city['city_id'].to_i` in `resolve_city`
   - `zone['zone_id'].to_i` in `resolve_zone`
   - `area['area_id'].to_i` in `resolve_area` (when present)

### Files to Change
- `backend/engines/spree_pathao_courier/app/services/spree/pathao_courier/error.rb`
- `backend/engines/spree_pathao_courier/app/services/spree/pathao_courier/address_resolver.rb`

### Fix Applied (2026-07-06)

**error.rb** — Added missing class:
```ruby
class AddressNotFoundError < Error; end
```

**address_resolver.rb** — Added `.to_i` conversions to prevent TypeError:
```ruby
{ city_id: city['city_id'].to_i, city_name: city['city_name'] }
{ zone_id: zone['zone_id'].to_i, zone_name: zone['zone_name'] }
{ area_id: area['area_id']&.to_i, area_name: area['area_name'] }
```

The safe navigation operator `&.to_i` on `area_id` handles the optional nil case.
